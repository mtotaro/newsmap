## Core architectural rules

1. **No hardcoded strings in components.** Every user-facing string goes in `messages/es.json` and `messages/en.json`. Use `useTranslations()` in client components, `getTranslations()` in server components.

2. **Cursor-based pagination only.** Never use `OFFSET`. The cursor is `published_at` of the last seen article. See `/api/feed/route.ts` for the reference implementation.

3. **Fetch per source, not per user.** The Inngest cron fetches each active source once every 15 minutes regardless of subscriber count. The feed query joins `user_subscriptions`.

4. **No `any` in TypeScript.** Strict mode is on. If a type is unknown, use `unknown` and narrow it.

5. **Tailwind v4 syntax.** No `tailwind.config.js`. Use `@theme` in `app/globals.css` for design tokens. Do not add arbitrary values — extend the theme instead.

6. **Image proxying via `next/image`.** All external images must go through Next.js Image Optimization. The `remotePatterns` in `next.config.ts` must be updated when adding new sources.

7. **Auth is server-side.** Use `createServerClient` from `/lib/supabase/server.ts` in Server Components and Route Handlers. Never expose `SUPABASE_SERVICE_ROLE_KEY` to the client.

8. **Inngest for all async work.** No `setTimeout`, no raw `setInterval`, no ad-hoc cron. All background work is an Inngest function.

9. **Drizzle for all DB queries.** No raw `supabase.from()` queries in app code — only in migrations and triggers. All queries go through Drizzle.

## How to add a new news source

1. Open `lib/db/seed.ts`
2. Add a new entry to the `SOURCES` array following the existing pattern:
   ```typescript
   {
     name: 'Source Name',
     countryCode: 'AR',
     region: 'latam',           // latam | north_america | europe | asia | africa
     language: 'es',            // ISO 639-1
     logoUrl: 'https://...',
     websiteUrl: 'https://...',
     feedSections: [
       { key: 'sports', url: 'https://...feed/sports', labelEs: 'Deportes', labelEn: 'Sports' },
       { key: 'politics', url: 'https://...feed/politics', labelEs: 'Política', labelEn: 'Politics' },
     ],
   }
   ```
3. Verify the RSS URLs load correctly (run `npx tsx lib/db/verify-feeds.ts` — see Skills below)
4. Update `next.config.ts` `remotePatterns` with any new thumbnail domains
5. Run `npm run db:seed` to insert in dev, or commit the seed file and let CI handle production

**Valid `section_key` values:** `sports` | `politics` | `economy` | `tech` | `culture` | `world` | `health` | `science` | `entertainment`

## How to add a new i18n string

1. Add the key to `messages/es.json` (base language — Spanish first)
2. Add the same key to `messages/en.json` with the English translation
3. Use in a **Server Component**: `const t = await getTranslations('namespace'); t('key')`
4. Use in a **Client Component**: `const t = useTranslations('namespace'); t('key')`
5. For plurals: `"{count, plural, one {# article} other {# articles}}"`

Never add a string to a component directly. If the key doesn't exist in both JSON files, TypeScript will error (next-intl enforces this with the generated types).

## Running DB migrations

```bash
# 1. Edit lib/db/schema.ts
# 2. Generate migration SQL
npm run db:generate        # drizzle-kit generate

# 3. Apply to local Supabase (dev)
npm run db:push            # drizzle-kit push

# 4. Apply to production (via migration file — do NOT use push in prod)
npm run db:migrate         # drizzle-kit migrate
```

Migration files live in `lib/db/migrations/`. Commit them — they are the source of truth for schema history.

## Sub-agent patterns for parallel work

When a task can be split into independent subtasks, spawn sub-agents with the `Agent` tool and collect results. This project has several natural parallelism points:

### Parallel RSS verification (35 sources)
Split sources into 3 batches by region and run 3 agents simultaneously. Each agent fetches and parses each feed, reports `{ name, url, status, has_thumbnail, encoding, section_feeds }`.

### Parallel component generation
When building multiple independent UI components (e.g., `ArticleCard` + `WorldMap` + `OnboardingScreen`), spawn one agent per component with the full spec. Merge results after.

### Parallel translation
When adding a large set of i18n keys, spawn one agent for `es.json` and one for `en.json` simultaneously. Merge and verify key parity.

### Parallel seed verification
After adding new sources, spawn agents per region to verify RSS URLs and thumbnail availability concurrently.

## Known gotchas

- **NYT RSS** is truncated — no full article content, just headline + 1 line. That's fine for the model.
- **Reforma (MX)** may require authentication — use general news RSS or skip if blocked.
- **WSJ / FT** excluded — require paid subscription for useful feeds.
- **Folha de SP, Le Monde, Der Spiegel, La Repubblica**: may send ISO-8859-1 despite declaring UTF-8. Run `/verify-feeds` after adding them.
- **Cloudflare Bot Fight Mode**: a few CDNs block repeated RSS fetches. The Inngest throttle (1 req/10min per domain) mitigates this. User-Agent must include contact email.
- **First subscriber on a new source**: the API route POST `/api/subscriptions` checks subscriber count and fires an immediate Inngest event if count was 0. Never remove this logic.
- **Supabase Realtime filter**: filter by `source_id=in.(...)` to avoid triggering the "N new articles" banner for unsubscribed sources.
